# AI Telegram Assistant — Центр Красок #1

MVP Telegram-бота с AI-ассистентом для компании [«Центр Красок #1»](https://centr-krasok.kz/) — интернет-магазин лакокрасочных и декоративных материалов в Казахстане (Алматы, Астана).

Пользователь общается с ботом как в обычном чате: без команд и меню, задаёт вопросы о компании и получает ответы на основе подготовленной базы знаний.

## Задача проекта

- Собрать и структурировать информацию о компании с официального сайта и открытых источников.
- Реализовать backend-сервис (Telegram-бот) с интеграцией внешнего LLM API.
- Ограничить «галлюцинации» модели: ответы только по базе знаний, фильтры и system prompt.
- Поддержать контекст диалога и базовую защиту от некорректных / посторонних запросов.

## Функциональность

| Возможность | Реализация |
|-------------|------------|
| Чат без команд | Обработка текстовых сообщений, приветствие на `/start` |
| База знаний | `data/company_knowledge.md` — услуги, контакты, бренды, клиенты |
| Контекст диалога | История последних N сообщений на пользователя (`context_manager.py`) |
| Снижение галлюцинаций | System prompt, `temperature: 0.3`, guardrails (off-topic, санитизация ответа) |
| Выбор LLM | Ollama (локально) или Google Gemini (облако) через `LLM_PROVIDER` |

**Примеры вопросов:** чем занимается компания, услуги, адреса салонов, бренды, клиенты, вакансии/партнёрство, контакты.

## Стек и инструменты

| Категория | Технология |
|-----------|------------|
| Язык | Python 3 |
| Telegram | [python-telegram-bot](https://github.com/python-telegram-bot/python-telegram-bot) 21.x |
| HTTP-клиент | [httpx](https://www.python-httpx.org/) (async) |
| Конфигурация | [python-dotenv](https://github.com/theskumar/python-dotenv) |
| LLM (на выбор) | [Ollama API](https://github.com/ollama/ollama) `/api/chat` или [Google Gemini API](https://ai.google.dev/) `generateContent` |
| Данные | Markdown-база знаний, собранная с centr-krasok.kz |

## Архитектура

```
Пользователь (Telegram)
        ↓
    bot.py — handlers, UX
        ↓
  guardrails.py — off-topic, проверка ответа
        ↓
  knowledge.py — system prompt + company_knowledge.md
        ↓
  ai_client.py — Ollama | Gemini
        ↓
  context_manager.py — история диалога
```

## Структура репозитория

```
├── data/company_knowledge.md   # База знаний о компании
├── src/
│   ├── bot.py                  # Telegram-логика
│   ├── ai_client.py            # Интеграция LLM
│   ├── knowledge.py            # Промпт и загрузка базы
│   ├── context_manager.py      # Контекст диалога
│   ├── guardrails.py           # Защита от выдумок и off-topic
│   └── config.py               # Настройки из .env
├── main.py
├── env.template                # Шаблон переменных (без секретов)
└── requirements.txt
```

## Источники данных

Информация подготовлена на основе:

- [centr-krasok.kz](https://centr-krasok.kz/) — главная, о магазине, контакты, новости
- Разделы для строителей и дизайнеров
- Публичные контакты и описание услуг компании

## Ограничения MVP

- Контекст хранится в оперативной памяти (сбрасывается при перезапуске)
- Актуальные цены и остатки не отдаются — только общая информация и ссылка на сайт
- Секреты (токены) хранятся локально в `.env`, в репозиторий не попадают
